# PRD — JSON 기반 회원정보 CRUD

## 1. 개요

JSON 파일을 영속 저장소로 사용하는 회원정보 관리 콘솔 애플리케이션을 C++20으로 구현한다.
외부 DB 없이 로컬 JSON 파일만으로 CRUD 전 기능을 검증하는 POC이다.

## 2. 목표

- nlohmann/json 라이브러리(NuGet)를 활용한 JSON 파일 I/O 패턴 확립
- Create / Read / Update / Delete 4가지 기능의 동작 검증
- 콘솔 메뉴 기반 인터랙션으로 각 기능을 직접 테스트

## 3. 사용자 스토리

| ID   | 스토리                                                          |
|------|----------------------------------------------------------------|
| US-1 | 사용자는 이름·이메일·전화번호를 입력해 새 회원을 등록할 수 있다.       |
| US-2 | 사용자는 전체 회원 목록을 조회할 수 있다.                            |
| US-3 | 사용자는 특정 회원 ID로 단건 상세 정보를 조회할 수 있다.              |
| US-4 | 사용자는 특정 회원 ID를 선택해 이름·이메일·전화번호를 수정할 수 있다.  |
| US-5 | 사용자는 특정 회원 ID를 선택해 회원을 삭제할 수 있다.                |
| US-6 | 모든 변경사항은 즉시 JSON 파일에 저장되어 재실행 후에도 유지된다.      |

## 4. 기능 요구사항

### 4-1. Create (회원 등록)

- 입력 항목: `name`, `email`, `phone`
- `id`는 현재 저장된 최대 id + 1로 자동 채번
- `created_at`은 실행 시점의 날짜(YYYY-MM-DD)를 자동 기록
- 등록 성공 시 부여된 id를 출력

### 4-2. Read (조회)

- **전체 목록**: 모든 회원을 id 오름차순으로 출력
  - 출력 컬럼: `id`, `name`, `email`, `phone`, `created_at`
- **단건 조회**: id 입력 → 해당 회원 상세 출력
  - 존재하지 않는 id 입력 시 오류 메시지 출력

### 4-3. Update (수정)

- id 입력 → 해당 회원 현재 값 표시 → 새 값 입력
- 각 필드는 빈 입력 시 기존 값 유지
- 수정 성공 시 변경된 내용 출력

### 4-4. Delete (삭제)

- id 입력 → 해당 회원 정보 표시 후 삭제 여부 확인(Y/N)
- 확인 후 삭제, 삭제 성공 메시지 출력
- 존재하지 않는 id 입력 시 오류 메시지 출력

### 4-5. 데이터 영속성

- 데이터 파일 경로: `data/members.json`
- 파일이 없을 경우 최초 실행 시 `{"members":[]}` 로 자동 생성
- 모든 CUD 연산 후 즉시 파일에 저장(flush)

## 5. 비기능 요구사항

| 항목       | 내용                                              |
|-----------|--------------------------------------------------|
| 언어       | C++20                                            |
| JSON 라이브러리 | nlohmann/json v3.11.3 (NuGet)              |
| 빌드 환경  | Visual Studio 2022, MSVC v145, x64               |
| 인코딩     | UTF-8 (한글 이름·출력 지원)                         |
| 오류 처리  | JSON 파싱 실패·파일 I/O 오류 시 예외 출력 후 계속 실행 |
| 입력 검증  | 필수 필드 공백 입력 방지 (name, email, phone)        |

## 6. 데이터 스키마

```json

  "members": [

      "phone": "010-1234-5678",
      "created_at": "2026-06-11"
    }
  ]
}
```
